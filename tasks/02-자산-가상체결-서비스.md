# 02. 자산 가상체결 서비스

## 개요
가상 매수/매도 체결 시 자산(현금, 보유 종목) 행을 일관되게 갱신하는 서비스를 구현한다.
동시 보유 1종목 정책과 즉시 체결(주문값=결과값) 정책을 서비스 레벨에서 강제한다.

## 테스트 대상 함수
- `apps/asset/services.py::get_cash_asset() -> Asset`
- `apps/asset/services.py::get_open_position() -> Asset | None`

- `apps/asset/services.py::apply_virtual_buy(stock_code: str, price: Decimal, quantity: int) -> tuple[Asset, Asset]`
- `apps/asset/services.py::apply_virtual_sell(stock_code: str, price: Decimal, quantity: int) -> tuple[Asset, Asset]`

## 진입점(있는 경우)
- 없음

## 비즈니스 규칙
1. 현금 row는 `stock_code=None`, `quantity=1`, `unit_price==total_amount` 규칙을 유지한다.
2. 매수 시 현금 `total_amount`를 차감하고 보유 종목 row를 생성 또는 갱신한다.
3. 매도 시 보유 종목 `quantity`를 차감하고 현금을 증가시킨다.
4. 보유 수량이 0이 되면 해당 보유 row를 삭제한다.
5. 종목코드는 `shared/stock_universe.py`의 지원 종목만 허용한다.
6. 동시 보유 1종목 정책: 보유 종목이 있을 때 다른 종목 매수는 금지한다.
7. 수수료/슬리피지/부분체결은 반영하지 않는다.

## 에러 케이스
- 지원하지 않는 종목코드
- 가격/수량이 0 이하
- 현금 부족 매수
- 보유하지 않은 종목 매도
- 보유 수량 초과 매도
- 현금 row 부재 또는 비정상 데이터(중복 현금 row 등)

## 참고
- DB 스키마: `apps/asset/models.py::Asset`
- 정책: 동시 보유 1종목, 가상 주문 즉시 체결
- 공통 모듈: `shared/stock_universe.py`
