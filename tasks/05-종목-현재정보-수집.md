# 05. 종목 현재정보 수집

## 개요
한국투자증권 `inquire-price` API로 종목 현재정보를 수집해 `MarketSnapshot`에 전체 필드를 저장한다.
트레이딩 프롬프트에서 사용하는 15개 필드 추출 함수도 함께 제공한다.

## 테스트 대상 함수
- `shared/external/kis_quote.py::fetch_inquire_price(stock_code: str) -> dict`
- `apps/market/services.py::normalize_market_snapshot(stock_code: str, payload: dict) -> dict`
- `apps/market/services.py::upsert_market_snapshot(snapshot_data: dict) -> MarketSnapshot`
- `apps/market/services.py::collect_market_snapshots(stock_codes: list[str] | None = None) -> dict`
- `apps/market/services.py::to_prompt_fields(snapshot: MarketSnapshot) -> dict`

## 진입점(있는 경우)
- management command: `collect_market_snapshot`
- 인자: 없음
- 옵션:
  - `--stock-codes` (선택, 쉼표 구분. 미지정 시 대상 10종목 전체)

## 비즈니스 규칙
1. 대상 10종목에 대해서만 수집한다.
2. API 응답의 전체 수집 필드를 `MarketSnapshot` 컬럼에 모두 저장한다.
3. `external_id`는 `MarketSnapshot.build_external_id(stock_code, published_at)`로 생성한다.
4. 동일 `external_id`는 upsert하여 idempotent하게 유지한다.
5. 숫자/날짜 문자열은 모델 타입에 맞게 변환한다.
6. 트레이딩 프롬프트용 15개 필드(`per`, `pbr`, `eps`, `hts_avls`, `hts_frgn_ehrt`, `frgn_ntby_qty`, `pgtr_ntby_qty`, `vol_tnrt`, `w52_hgpr`, `w52_lwpr`, `w52_hgpr_vrss_prpr_ctrt`, `w52_lwpr_vrss_prpr_ctrt`, `mrkt_warn_cls_code`, `invt_caful_yn`, `short_over_yn`) 추출 함수를 제공한다.

## 에러 케이스
- KIS 인증 실패 또는 API 오류 응답
- 필수 시각 값 누락으로 external_id 생성 불가
- 숫자/날짜 파싱 실패
- 지원하지 않는 종목코드 입력

## 참고
- DB 스키마: `apps/market/models.py::MarketSnapshot`
- 외부 API: `https://openapi.koreainvestment.com:9443/uapi/domestic-stock/v1/quotations/inquire-price`
- 환경변수: `KIS_APP_KEY`, `KIS_APP_SECRET`, `KIS_HTS_ID`, `KIS_ACCT_STOCK`, `KIS_PROD_TYPE`
