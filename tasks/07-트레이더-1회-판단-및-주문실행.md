# 07. 트레이더 1회 판단 및 주문 실행

## 개요
자산 상태를 기준으로 BUY/HOLD 또는 SELL/HOLD를 1회 판단하고, 판단 이력/주문 이력을 저장한다.
LLM 실패 시 기본 HOLD 처리와 에러 이력 기록을 보장한다.

## 테스트 대상 함수
- `apps/trader/services.py::build_buy_prompt(now: datetime | None = None) -> str`
- `apps/trader/services.py::build_sell_prompt(stock_code: str, now: datetime | None = None) -> str`
- `apps/trader/services.py::record_decision_history(request_payload: str, response_payload: str, parsed_decision: dict, processing_time_ms: int, is_error: bool, error_message: str | None) -> DecisionHistory`
- `apps/trader/services.py::execute_buy(decision_history: DecisionHistory, stock_code: str, price: Decimal, quantity: int) -> OrderHistory`
- `apps/trader/services.py::execute_sell(decision_history: DecisionHistory, stock_code: str, price: Decimal, quantity: int) -> OrderHistory`
- `apps/trader/services.py::run_trading_cycle(now: datetime | None = None) -> DecisionHistory`

## 진입점(있는 경우)
- management command: `run_trader_once`
- 인자: 없음
- 옵션: 없음

## 비즈니스 규칙
1. 보유 종목이 없으면 매수 판단, 보유 종목이 있으면 해당 1종목 매도 판단을 수행한다.
2. 매수 프롬프트는 대상 10종목 정보를 모두 포함한다.
3. 매도 프롬프트는 현재 보유 1종목 정보만 포함한다.
4. 프롬프트에는 현재시각, 자산 잔액, 최근 10분 Redis 가격 흐름, 최신 시장정보(15개 필드), 최근 7일 공시, 최근 10건 뉴스(`useful=true` 또는 `useful is null`)와 수집시각을 포함한다.
5. LLM 응답 파싱 실패/타임아웃/빈 응답은 기본 `HOLD`로 처리한다.
6. `DecisionHistory`는 성공/실패와 무관하게 항상 저장한다.
7. `BUY`/`SELL`일 때만 주문을 실행하고 `OrderHistory`를 저장한다. `HOLD`는 주문 이력을 만들지 않는다.
8. 가상 주문은 즉시 체결로 처리하여 `order_*`와 `result_*`를 동일 값으로 저장한다.
9. 주문 실행은 자산 서비스(`apply_virtual_buy`, `apply_virtual_sell`)를 사용해 잔고를 갱신한다.
10. 결과값은 `DecisionHistory.result`의 enum(`BUY`, `SELL`, `HOLD`)과 일치해야 한다.

## 에러 케이스
- LLM 호출 실패/응답 파싱 실패
- `decision.result`가 허용값이 아닌 경우
- BUY인데 종목코드/가격/수량 누락 또는 0 이하
- SELL인데 미보유 종목 또는 수량 초과
- 보유 종목이 있는데 BUY 시도, 미보유 상태에서 SELL 시도
- 프롬프트 컨텍스트 데이터(뉴스/공시/시세) 일부 누락
