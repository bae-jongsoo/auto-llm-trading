# 04. 다트 공시 수집

## 개요
OpenDartReader `list`를 사용해 대상 종목 공시를 수집하고 `external_id` 기반 upsert를 수행한다.
종목코드와 DART corp_code 매핑은 고정 10종목 기준으로 관리한다.

## 테스트 대상 함수
- `shared/stock_universe.py::resolve_target_corp_codes(stock_codes: list[str] | None = None) -> dict[str, str]`
- `shared/external/dart_api.py::fetch_disclosures(corp_code: str) -> list[dict]`
- `apps/dart/services.py::upsert_disclosures(stock_code: str, corp_code: str, rows: list[dict]) -> list[DartDisclosure]`
- `apps/dart/services.py::collect_dart(stock_codes: list[str] | None = None) -> dict`

## 진입점(있는 경우)
- management command: `collect_dart`
- 인자: 없음
- 옵션:
  - `--stock-codes` (선택, 쉼표 구분. 미지정 시 대상 10종목 전체)

## 비즈니스 규칙
1. 수집 대상은 사전 지정 10종목으로 제한한다.
2. 종목코드별 `corp_code`를 사용해 OpenDartReader `list`를 호출한다.
3. `DartDisclosure.build_external_id(corp_code, rcept_no)`로 외부키를 생성한다.
4. `external_id` 기준 upsert로 idempotent하게 저장한다.
5. 공시 본문 설명(description)이 없는 경우 `None` 허용한다.
6. 공시 시각(`published_at`)은 가능하면 원본 값을 저장하고, 없으면 `None` 허용한다.

## 에러 케이스
- `DART_API_KEY` 누락
- OpenDartReader 호출 실패
- `rcept_no` 누락 등으로 external_id 생성 불가
- 종목코드-corp_code 매핑 누락
- 지원하지 않는 종목코드 입력

## 참고
- DB 스키마: `apps/dart/models.py::DartDisclosure`
- 외부 API: OpenDartReader `list`
- 환경변수: `DART_API_KEY`
- 대상 종목/법인코드: PROJECT_RULES의 대상 종목 표
