# 03. 뉴스 수집 및 요약

## 개요
네이버 뉴스 API로 대상 종목 뉴스를 수집하고, 기사 본문 크롤링 + LLM 요약/유용성 판정을 저장한다.
수집은 `external_id` 기준 idempotent upsert로 처리한다.

## 테스트 대상 함수
- `shared/external/naver_news.py::fetch_news(stock_code: str, limit: int = 10) -> list[dict]`
- `shared/external/web_content.py::extract_article_text(url: str) -> str`
- `apps/news/services.py::upsert_news_items(stock_code: str, items: list[dict]) -> list[News]`
- `apps/news/services.py::summarize_news(news: News) -> News`
- `apps/news/services.py::collect_news(stock_codes: list[str] | None = None, limit: int = 10) -> dict`

## 진입점(있는 경우)
- management command: `collect_news`
- 인자: 없음
- 옵션:
  - `--stock-codes` (선택, 쉼표 구분. 미지정 시 대상 10종목 전체)
  - `--limit` (선택, 종목당 수집 건수. 기본 10)

## 비즈니스 규칙
1. 종목코드는 대상 10종목만 허용한다.
2. 뉴스 저장 시 `News.build_external_id(stock_code, link)`를 사용한다.
3. `external_id` 기준으로 upsert하여 중복 저장을 방지한다.
4. 저장 후 `link` 본문 크롤링에 성공하면 뉴스 요약 프롬프트로 `ask_llm` 호출 후 `summary`, `useful`를 갱신한다.
5. 크롤링 실패 시 `summary=description`, `useful=None`으로 저장한다.
6. LLM 응답은 `{"summary": "...", "useful": true}` 형식을 기대한다.
7. published_at/created_at은 timezone-aware(KST 기준 운영)로 처리한다.

## 에러 케이스
- 네이버 API 호출 실패 또는 비정상 응답
- 기사 본문 크롤링 실패
- LLM 타임아웃/파싱 실패/빈 응답
- API 아이템의 필수 키(link 등) 누락
- 지원하지 않는 종목코드 입력

## 참고

- 외부 API: `https://openapi.naver.com/v1/search/news.json`
